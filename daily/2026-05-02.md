# 今日任务：给现有接口补第一个 pytest

日期：2026-05-02

## 今日目标
给当前 FastAPI 项目补上第一个可运行的 `pytest` 接口测试，先把“改完接口后能快速验证有没有坏”这件事建立起来。

## 为什么做这个
这周你已经连续碰了接口、Pydantic 和日志，下一步最该补的是最小测试闭环。对 Agent 工程来说，接口经常会随着提示词、schema、返回格式一起变化，如果没有一个能快速跑通的最小测试，后面接 tool schema 和最小 tool-calling flow 时会很容易改一处坏一处。今天是周六，任务可以轻一点，但这个轻任务很值。

## 任务说明
1. 选一个你现在已经能跑通的接口来写第一个测试，优先选 `GET /health`；如果你已经完成了 `POST /task`，也可以直接测它。
2. 使用 `pytest` 搭一个最小测试文件，目标只做一件事：发起请求并断言状态码和关键返回字段。
3. 如果仓库里还没有测试客户端相关依赖或基础写法，先用最简单、最容易跑通的方案，不追求抽象复用。
4. 测试名、断言和返回字段命名尽量写清楚，让你下周回来看时一眼能知道这个接口是否还正常。
5. 在 `README.md` 补一小段测试运行说明，至少写清楚如何执行这个测试；如果你前几天的 README 还没补完整，也只补和今天测试直接相关的最小内容。
6. 如果你前面的 `/task`、日志或模型还没完全做完，今天也可以先对一个更简单的现有接口补测试，重点是先把 `pytest` 跑起来。

## 验收标准
1. 仓库里至少新增 1 个可运行的 `pytest` 测试。
2. 该测试能断言接口状态码和至少 1 个关键 JSON 字段。
3. 本地执行测试命令时，至少有 1 条测试通过。
4. `README.md` 至少补清楚 1 种运行测试的方式。
5. 整体改动控制在 20 到 60 分钟内可完成。

## 推荐目录/文件
- `tests/`
- `app/main.py`
- `README.md`
- `pyproject.toml` 或 `requirements*.txt`（如果你需要补测试依赖说明）
