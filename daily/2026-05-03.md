## 任务说明
1. 打开当前项目的 `README.md`，先按“新同事第一次接手这个仓库”的视角检查：现在有没有写清楚环境准备、安装依赖、启动服务、运行测试、验证接口。
2. 至少补齐 4 块最小信息：
   - 项目是做什么的
   - 本地如何启动 FastAPI 服务
   - 本地如何运行当前已有的 `pytest`
   - 如何手动验证一个现有接口，例如 `GET /health` 或 `POST /task`
3. 如果你这几天已经加了日志或 Pydantic 模型，可以在 README 里顺手补 2 到 4 句非常简短的说明，写清楚这些能力现在落在哪个接口或文件上，但不要把 README 写成长文档。
4. 如果当前目录结构已经比较清楚，补一个最小“项目结构”小节，只列最关键的几个目录或文件，例如 `app/`、`tests/`、`daily/`。
5. 检查命令示例的可执行性，尽量统一成一种风格，不要一会儿是伪命令、一会儿是口头描述。
6. 如果你发现 README 里有和当前代码状态不一致的地方，优先修正不一致，而不是新增很多背景介绍。
7. 如果你前几天的 pytest 还没补完，今天也可以先把“如何运行测试”写成占位说明，并明确当前已经能验证什么、还不能验证什么。

3. README 内容和当前项目现状基本一致，没有明显过时描述。
4. 整体整理工作控制在 20 到 60 分钟内可完成。
