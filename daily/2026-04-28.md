# Day 003 / 2026-04-28

## 今日目标
为当前 FastAPI 小项目增加一个最小可用的 `GET /version` 接口，返回固定版本信息，并让项目的运行说明再清楚一点。

## 为什么做这个
`/version` 是比 `/health` 稍微更接近真实服务的一步。它不复杂，但能帮你继续熟悉路由定义、JSON 返回结构、项目入口位置，以及“改完后要能自己验证”的基本节奏。后面做 Pydantic、日志、测试和 tool schema 时，也会更自然地基于这类稳定的小接口往前推。

## 任务说明
1. 在现有 FastAPI 应用里新增一个 `GET /version` 路由。
2. 先不要接配置文件或环境变量，直接返回固定版本值即可，例如一个简单的版本号字符串。
3. 如果你昨天还没完成 `/health`，今天可以先补一个最小可启动的应用入口，再继续加 `/version`；但今天的主要验收对象仍然是 `/version`。
4. 顺手检查 `README.md` 里的运行说明是否够清楚。
5. 如果 README 里还没有“如何验证 `/version`”的说明，就补一小段启动和访问方式。

注意：
- 不要扩展成复杂业务接口。
- 不要引入数据库、配置中心或额外框架。
- 不要在今天把 Pydantic、日志、pytest 一次性全塞进来，先把接口链路做干净。

## 验收标准
1. 本地可以启动 FastAPI 服务。
2. 访问 `GET /version` 时，能稳定返回一个固定版本信息。
3. README 至少写清楚启动命令，以及如何验证 `/version`。
4. 整体改动控制在 20 到 60 分钟内可完成。

## 推荐目录/文件
- `app/`
- `app/main.py`
- `README.md`
- `tests/`（今天可以先不写测试，但可以想一下后面怎么测这个接口）
