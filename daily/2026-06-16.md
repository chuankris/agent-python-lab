# 今日任务：落地第一个最小 FastAPI 练习

日期：2026-06-16

## 今日目标
把仓库里第一份真实可运行的 Python 练习落到 `exercises/` 下，完成一个最小 FastAPI 骨架，并预留后续放 Pydantic、日志、测试和 tool schema 的位置。

## 为什么做这个
你现在这个仓库里的任务文档已经连续产出了不少，但代码目录还是空的。继续往后做 `/task`、tool schema、日志这些题目之前，先把“能放代码、能启动、能测试”的最小练习骨架搭起来，收益最高。

这一步很像你以前做 Java 服务时先把工程骨架立住：目录清楚、入口明确、测试有位置、README 能告诉别人怎么跑。Agent 工程后面会比这个复杂很多，但底层仍然是这套最小工程纪律。

如果你前几天没跟上，也没关系。今天这题就是重新落地起点，不依赖之前已经写过任何接口。

## 任务说明
1. 在 `exercises/` 下新建一个练习目录，建议叫 `day-001-project-skeleton`，名字可以按你自己的习惯微调，但要一眼看出这是第一个 FastAPI 练习。
2. 在这个练习目录里至少准备出这些内容：`app/`、`tests/`、`README.md`，以及一个 Python 入口文件。
3. 入口文件里实现一个最小可启动的 FastAPI 应用；如果你愿意顺手多走半步，可以直接把 `GET /health` 一起做掉，但不要扩展第二个接口。
4. 为后续学习提前留出位置：可以先放空文件、占位说明或最小注释，预留 `schemas.py`、`logging.py`、`tool_schema.py` 这类文件入口，但今天不要把它们都实现完。
5. 在 `tests/` 里补 1 个最小测试占位，优先测应用是否能导入，或者 `/health` 是否返回预期；只要有 1 个能跑通的小测试就够。
6. 给这个练习目录写一个短 README，至少写清楚：怎么创建虚拟环境、怎么安装依赖、怎么启动、怎么跑测试。
7. 今天不要求接模型、不要求做真实 tool calling、不要求补完整日志体系；重点是把后面所有任务的“施工地基”搭出来。

## 验收标准
1. `exercises/` 下出现了一个新的练习目录，且包含 `app/`、`tests/`、`README.md`。
2. FastAPI 应用入口已经存在，并且本地有明确启动方式。
3. 至少有 1 个最小 `pytest` 用例或测试占位被补上，且你知道怎么执行它。
4. README 至少写清楚依赖安装、启动命令、测试命令这 3 件事。
5. 整体工作量控制在 20 到 60 分钟，没有顺手把 `/task`、多工具编排、复杂日志全部做进去。
