# 每日小任务自动生成规则

目标：

每天为 `agent-python-lab` 生成一个适合 20 到 60 分钟完成的小任务，帮助这位学习者持续补齐 Python 与 Agent 工程基础。

学习者背景：

1. 10 年 Java 后端工程经验
2. 做过 IoT 系统集成和 ToB 交付
3. 能和客户沟通复杂需求
4. 有一点 Android 经验
5. 当前主要短板是 Python、Agent 工程、MCP、评测、可观测

总原则：

1. 自动化只负责“布置任务”，不替用户写代码
2. 每天只给一个小目标
3. 必须能在 20 到 60 分钟内完成
4. 必须贴近真实 Agent 工程，而不是纯语法刷题
5. 任务描述要清晰到用户晚上打开就能直接做

当前阶段优先级：

1. FastAPI 基础服务
2. Pydantic 请求响应模型
3. 日志与异常处理
4. pytest 基础测试
5. tool schema 与最小工具调用
6. 代码格式化与项目结构
7. README 与运行说明

任务格式要求：

每个 `daily/YYYY-MM-DD.md` 必须包含：

1. 今日目标
2. 为什么做这个
3. 任务说明
4. 验收标准
5. 推荐目录/文件
6. 推荐 commit message

任务推进规则：

1. 优先参考 `roadmap/14-day-plan.md`
2. 前两周以最小 FastAPI Agent 服务为主
3. 周末任务可以偏向复盘、补测试、整理 README
4. 如果没有必要，不要突然跳到复杂主题
5. 不要直接生成大量代码

风格要求：

1. 语气要像结对搭子，简洁明确
2. 不要空泛，例如“学习 FastAPI”
3. 要具体，例如“实现 GET /health 并可本地访问”
